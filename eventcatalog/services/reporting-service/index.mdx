# Reporting Service

The Reporting Service provides analytics and business intelligence by consuming events across the BioPro platform and generating insights, reports, and dashboards.

## Responsibilities

- Generate manufacturing reports
- Track product changes and trends
- Provide operational analytics
- Create executive dashboards
- Support regulatory reporting requirements

## Events Consumed

### ApheresisPlasmaProductCreated
**From**: manufacturing-service
**Purpose**: Generate manufacturing reports
**Critical**: No
**Handler**: `ProductCreatedAnalytics`

### ApheresisPlasmaProductUpdated
**From**: manufacturing-service
**Purpose**: Track product changes
**Critical**: No
**Handler**: `ProductUpdatedAnalytics`

## Data Products Generated

- **Daily Manufacturing Summary** - Products created, completed, updated
- **Quality Metrics Dashboard** - Pass/fail rates, compliance status
- **Distribution Analytics** - Delivery times, temperature monitoring
- **Inventory Reports** - Stock levels, product aging, expiration tracking
- **Regulatory Reports** - FDA submissions, audit trails

## Technical Details

### Technology Stack
- **Runtime**: Python 3.11
- **Processing**: Apache Spark
- **Messaging**: Apache Kafka with Avro
- **Database**: PostgreSQL (OLTP), Snowflake (OLAP)
- **Visualization**: Tableau, Grafana

### Deployment
- **Environment**: Kubernetes
- **Replicas**: 2
- **Port**: 8087

### Data Pipeline
- Real-time event streaming from Kafka
- Batch processing for historical analytics
- Data warehouse updates every 15 minutes
- Dashboard refresh every 5 minutes
